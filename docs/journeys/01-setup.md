# Installation
First time setup is important.

If Tim wasn't the owner of this, he's basically ambivelent. If it's interesting enough, he'll clone a repo,
build and try it out. Lily will do a lot of work, but a lot of "dev setup" processes get complex, she'd need
an agent to help her out. That's hours spent. Daniel, however, does not give a shit. If there's no installer,
it's not getting installed. Ganesh, otoh, just wants to know how updates will be rolled out.

Environment complexity is mostly consistent. More prerequisites = harder to install. Rust is great, across
the board, unless we're saying you have to install cargo. Also, Ganesh actively makes it harder for employees
at his company to install either cargo or node. Dependencies are risk to him.


# Journey: Lily sets up an agent
She's done it before with open-strix and OpenClaw. She just wants to setup an agent, name it, give it a purpose
and get it growing.

open-strix and OpenClaw both start with a Git-backed directory where the agent "lives", so that's what she's
expecting. She also cares a lot about which model backs the agent and what it costs. She's a hawk for reducing
costs (while also enjoys chatting with agents).

If she's not told what the setup process is, she'll just figure it out. She's driven. But also, the less she has
to figure out the better. If anything gets hard, an agent can help (better if an agent can do it). Her eyes
drift immediately toward the "new agent" button, but then is met with a moment of overwhelm as all the options
rush toward her. She's got her agent to help, she'll get through this.


# Journey: Daniel setting up Claude Code
He's not interested in a chat partner. These things cost money! Besides, he's got a job to do.

He opens Elanus UI for the first time. Somewhat bewildered, he's just trying to figure out how to add Claude Code
to the agent system. Also, why is he even doing this? He looks for interesting features that might draw his interest,
secure his dedication. Billing makes sense. He sees that Elanus helps with predicting and controlling costs.
That suits him for now, he presses on.

After setting up Claude Code, he's somewhat annoyed because _(literally anything here)_ doesn't work quite right
and after banging on it for a few minutes, he decides to quit using Elanus and just use Claude Code normally.
